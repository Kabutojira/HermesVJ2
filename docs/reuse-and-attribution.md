# Reuse, licensing, and attribution

## Project license

HermesVJ2 is currently all rights reserved; see `LICENSE`. Publication on GitHub is not an open-source license grant.

## Source inspiration

HermesVJ2 is inspired by https://github.com/Kabutojira/HermesVJ. It reinterprets the source project's generative-art gallery purpose, dark luminous mood, patient pacing, and repository-driven publishing. It does not embed the source application's p5.js/Hydra iframe runtime, sketch manifest, media, or authored visual assets.

Because HermesVJ has no detected license, contributors must treat its code and assets as unavailable for copying unless the copyright holder supplies written authorization. Ideas may be reimplemented independently; source code, images, audio, shaders, and authored text must not be copied by default.

## Current assets

The production world uses procedural Three.js geometry and materials. Public SVG metadata artwork in `public/` was authored for HermesVJ2. No remote runtime artwork, stock media, font file, audio file, or source-repository visual asset is required.

## Third-party software

Runtime and development dependencies are distributed under their own licenses and are not relicensed by this repository. The principal runtime dependencies are React, Three.js, React Three Fiber, Drei, Zustand, and postprocessing. `package-lock.json` pins the resolved dependency graph; consult each package's installed license before redistribution.

## Contribution rule

Every new asset must record creator, source URL (if any), license, and modifications in this document or a chapter-local attribution file. Do not use an asset when its redistribution or derivative-work rights are unclear.
